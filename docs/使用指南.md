# 使用指南

本文档提供所有命令的速查、预期输出、参数说明及常见问题排查。

---

## 命令速查

### 数据获取

```bash
python data_fetcher.py                  # 增量更新（缓存 ≤7 天则跳过）
python data_fetcher.py --force          # 强制重新下载
python data_fetcher.py --max 5          # 只下载前 5 个标的（测试用）
python data_fetcher.py --backend alpha_vantage  # 切换后端（兜底用）
```

**说明：** 下载日线行情并缓存为 `data/market_data.parquet`（parquet 格式）。7 天内重复运行跳过下载。

> ⚠️ 旧入口，日常使用不需要单独运行——`train_model.py` 会自动调取缓存数据。

---

### 因子计算

```bash
python factor_engine.py                 # 计算全量因子（需先有行情数据）
python factor_engine.py --cache         # 使用缓存的行情数据计算因子
```

**说明：** 读取行情 → 计算 9 个技术因子 → 因子验证报告 → 保存因子验证到 `data/factor_validation.csv`。

**因子清单（9 个）：**

| 类别 | 因子 |
|:---|---:|
| 动量类 | MOMO_20, MOMO_60, MOM_RATIO |
| 均值回归类 | RSI_14, VOL_MA_RATIO |
| 波动率类 | VOLATILITY_20, BB_WIDTH |
| 风险类 | MAX_DD_60 |
| 资金流类 | CHAIKIN_MF |

输入 OHLCV → 输出 9 个因子值，每个因子为纯函数，可独立测试。

> ⚠️ 旧入口，日常使用不需要单独运行——`train_model.py` 会自动计算因子。

---

### 模型训练

```bash
python train_model.py                   # 完整 Walk-Forward 训练（全量 28 只）
python train_model.py --quick           # 快速模式（5 个标的，验证流程用）
python train_model.py --version V3      # 指定模型版本名（保存到 models/）
python train_model.py --target 21       # 预测 forward_return_21（默认 10）
python train_model.py --universe 28     # 只训练前 28 个标的
python train_model.py --load <exp_id>   # 查看已有实验摘要
```

**说明：** 完整流程：加载缓存数据 → 计算 9 个因子 → 横截面特征（rank + zscorre，共 27 特征）→ 缩尾 → Walk-Forward 窗口 → LightGBM 训练 → 保存实验。

**训练算法：** LightGBM + Walk-Forward 验证（6 年训练 → 1 年验证 → 1 年步进 → 1.5 年测试），3 个滚动窗口，早停 500 轮。

**输出：** `experiments/<exp_id>/` → config.json, summary.json, windows_metrics.csv, feature_importance.csv, OOS/测试预测, 模型文件。

### 因子探索

```bash
python factor_hunt.py --list                          # 列出所有候选因子
python factor_hunt.py BB_WIDTH                        # 测试单个因子（默认 forward_return_10）
python factor_hunt.py BB_WIDTH --period 21            # 测试因子在 21 日周期上的表现
python factor_hunt.py BB_WIDTH HIGH_LOW_RATIO --all   # 批量测试多个因子
```

**说明：** 以 IC/ICIR/层组合单调性为筛选标准测试新因子。结果记录在 `notes/因子寻找.md`。

---

### 回测

```bash
python run_backtest.py                # 列出所有可用实验
python run_backtest.py <exp_id>       # 对指定实验运行回测
python run_backtest.py <exp_id> --top 10      # 覆盖 top_k 参数（默认 2）
python run_backtest.py <exp_id> --no-regime   # 关闭 SPY MA200 市场状态过滤器
```

**说明：** 加载指定实验的 OOS 预测 → 生成调仓信号 → vectorbt 回测 → 输出指标 + 图表。

**回测配置：** vectorbt Portfolio.from_signals，入场次日开盘，最小持仓 2 天，最大 30 天，单边费率 0.17%（佣金 0.1% + 滑点 0.05% + 点差 0.02%）。

**输出：** `experiments/<exp_id>/backtest_report/` → report.json（指标）, report.png（权益曲线 + SPY 基准对比 + 回撤图）。

---

### 每日信号生成（Pipeline 核心）

```bash
python scripts/daily_inference.py                        # 默认：V2 模型（118 只），生成信号
python scripts/daily_inference.py --model V1             # 用 V1 模型（28 只，更保守）
python scripts/daily_inference.py --dry-run              # 预览信号但不保存文件
python scripts/daily_inference.py --no-regime            # 关闭 SPY MA200 市场状态过滤器
python scripts/daily_inference.py --top-k 10             # 每期最多买入 10 只（默认 5）
python scripts/daily_inference.py --date 2026-07-07      # 指定日期回测
python scripts/daily_inference.py --force-fetch          # 强制重新下载行情
```

**说明：** 10 步 Pipeline：加载模型 → 拉取数据 → 匹配标的池 → 计算 13 因子 → 横截面特征 + 缩尾 → LightGBM 推理 → 生成调仓信号 → 入场/出场信号 → 保存到 `data/signals/`。

**输出：**
- `data/signals/latest_entries.parquet` — 入场信号矩阵
- `data/signals/latest_exits.parquet` — 出场信号矩阵
- `data/signals/latest_predictions.parquet` — 预测值
- `data/signals/latest_summary.json` — 摘要统计

> 🎯 **日常使用入口脚本。** 模拟盘和未来实盘均以此脚本生成信号。

---

### 模拟盘运行

```bash
python scripts/run_simulation.py                           # 默认运行（V1 模型，$10,000 初始资金）
python scripts/run_simulation.py --model V2                # 切换 V2 模型（118 只）
python scripts/run_simulation.py --initial-balance 50000   # 指定初始资金
python scripts/run_simulation.py --no-inference            # 不重新生成信号（使用已有信号文件）
python scripts/run_simulation.py --scale 3                 # 3x 杠杆（安全范围内）
python scripts/run_simulation.py --top-k 10                # 每期最多买入 10 只
```

**说明：** 每日自动交易循环：
1. 调用 `daily_inference` 生成信号
2. 加载/恢复模拟账户状态（从 `data/simulation_state.json`）
3. 处理出场信号 → 风控检查 → 入场信号
4. 硬止损检查
5. 输出每日摘要
6. 保存状态

**风控规则：** 杠杆上限 7x（可调），每日最多 4 笔交易，单笔硬止损 8%，单标的最大暴露 30% of equity。

**输出：**
- `data/execution_audit.csv` — 每笔交易的审计日志
- `data/simulation_state.json` — 状态快照（状态可恢复）
- 终端每日摘要（持仓明细、盈亏、风控触发）

> ⚠️ **Broker(simulate=True)**，纯本地模拟，不需要 MT5 账户或 API Key。

```bash
# 2. 回测验证

# 3. 用新模型生成信号
python scripts/daily_inference.py --model V3
```

## 输出解读

### 数据完整性报告

```
检查: OK=22(85%), 需关注=4/26
  [V] 数据偏少: 420天 (<200天，无法算 MA200)
  [BRK-B] 缺失值: 45个(2.3%)
```

- **数据偏少**：该股票上市较晚，不足 200 天。可从 `src/universe.txt` 移除，或不依赖它的 MA200 因子。
- **异常波动 >±20%**：可能是财报跳空或拆股。少量可接受，大量需检查数据。

### 因子验证

```
[AAPL] MOMO_20: 范围异常 [-5.23, 0.45] (应在 [-0.5, 0.5])
```

通常源于：财报日跳空、拆股未还原、数据源错误。在 Python 中定位排查：

```python
from src.data.fetcher import fetch_all_data
df = fetch_all_data()
aapl = df.xs("AAPL", axis=1, level=0)
aapl["Close"].pct_change(20).hist(bins=50)  # 看分布
```

```bash
python -c "import pandas as pd; print(pd.read_csv('data/execution_audit.csv'))"
```

审计日志列：`timestamp, ticket, symbol, side, volume, price, reason, status, pnl, error`

## 常见问题

### Q1: yfinance 报 "Too Many Requests / Rate limited"

**原因：** 代理未开启，或代理端口填错。
**解决：**
1. 确认 v2rayN/Clash 正在运行
2. 在 `.env` 中确认 `HTTP_PROXY` 端口与代理软件一致
3. 测试代理是否通：
   ```bash
   curl -x http://127.0.0.1:10808 https://www.google.com -I
   ```

### Q2: 不想用代理，能用 AV 吗？

能，但免费层 AV 只返回最近 100 天数据，不足以计算 200 日均线因子。仅作兜底：

```ini
# .env
DATA_BACKEND=alpha_vantage
```

### Q3: "ModuleNotFoundError: No module named 'quant'"

从**项目根目录**运行脚本，Python 才能找到 `src/` 包：

```bash
cd d:/Code_store/Quant
python data_fetcher.py
```

或永久安装到环境：

```bash
pip install -e .
```

### Q4: 模拟盘输出乱码

**原因：** Windows 中文环境 (GBK) 不支持 emoji 字符（✅ ❌ 🛑）。
**解决：** 已自动修复。`src/__init__.py` 中强制 stdout/stderr 使用 UTF-8 编码，所有脚本默认生效。

如果仍有乱码，手动设置环境变量：

```bash
PYTHONIOENCODING=utf-8 python scripts/run_simulation.py
```

### Q5: 因子全是 NaN

某股票数据不足 60 天会被跳过。检查 `data/factor_validation.csv`，从 `src/universe.txt` 移除问题标的。

### Q6: 模拟盘状态下如何重置？

删除状态文件和审计日志即可重新开始：

```bash
rm data/simulation_state.json
rm data/execution_audit.csv
```
