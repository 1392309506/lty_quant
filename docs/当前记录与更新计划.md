## 二、实验结论

### 实验 1：不考虑最大回撤的年化上限

旧模型 (28只, fwd_21, 30特征)，逐步放大杠杆：

| 配置 | 年化收益 | 夏普 | 最大回撤 | 满足 30%? |
|:---:|:---:|:---:|:---:|:---:|
| p=50 (基准) | 64.75% | 5.80 | -19.17% | ✅ |
| p=200 | 156.82% | 9.53 | -27.23% | ✅ |
| p=500 | 346.46% | 17.80 | -23.74% | ✅ |
| p=1000 | 591.44% | 27.75 | -25.67% | ✅ |
| p=1500 | 139.45% | 0.10 | -192.63% | ❌ |

**结论**: 旧模型信号质量高，杠杆放大后回撤锁定在 -19%~-28%（未同步放大）。理论年化上限 ~591%。**但高杠杆在真实交易中不可行**，见风险警示章节。

## 三、风险警示

### 高杠杆 ≠ 安全回撤

回测中的 -30% 回撤在真实杠杆交易中意味着极高的爆仓风险：

以 `p=50, top_k=5` 为例：
- `position_size = 1/5 × 50 = 10` → 每个仓位 = 账户的 **10 倍**
- 同时最多 5 个仓位 → **总敞口 50 倍**
- **1 个仓位反向 10%** = 账户归零
- **2% 反向** = 亏损 20%

真实交易中：
- Exness CFD 保证金要求通常 1%~2%
- 亏损达到保证金的 50%~80% 即触发强制平仓
- **实际可承受亏损可能只有账户的 1%~2%**

**安全杠杆估算**: 若单笔最大亏损控制在 2%，`top_k=5` 时 `position_scale` 应设为 0.1~1.0，而非当前的 50。

**Fees=0 问题**: vectorbt 费用模型按固定金额计算而非杠杆后金额，高杠杆下实际费用被低估，真实收益会低于回测。

## 四、模型训练参数总览

### 训练前设定的参数（决定模型能力上限）

| 参数 | 当前值 | 可选范围 | 说明 |
|:---|:---:|:---|:---|
| **预测目标** | forward_return_10 | 5天 / 10天 / 21天 | 预测未来的窗口长度 |
| **因子组合** | 10个因子→30特征 | 可增删任意因子 | 模型"看什么"做预测 |
| **Walk-Forward 窗口** | 6y+1y+1y 步进 | 可自由调整 | 训练/验证/测试切分 |
| **LGBM 学习率** | 0.05 | 0.01~0.1 | 收敛速度与过拟合 |
| **LGBM 叶子数** | 31 | 15~127 | 模型复杂度 |
| **标的池** | 28 / 118 只 | 任意 ticker 列表 | 模型覆盖范围 |
| **缩尾阈值** | 5σ | 3~10σ | 极端值处理 |

### 训练后可调参数（同一模型不同回测）

| 参数 | 说明 | 效果 |
|:---|:---|:---|
| **position_scale** | 杠杆倍数 | 收益和回撤同步放大 |
| **top_k** | 每期买入数 | 高→分散但信号稀释 |
| **min_hold / max_hold** | 持仓天数 | 影响交易频率 |
| **max_rebalances_per_week** | 每周调仓次数 | 影响交易成本 |
| **regime_filter** | 市场趋势过滤 | 牛熊市开关 |

### 核心自由参数

1. **预测目标**（5/10/21天）— 先决定，需重新训练
2. **因子组合**（选哪些因子）— 决定模型信号，需重新训练
3. **杠杆+持仓数**（position_scale × top_k）— 决定最终风险收益，无需重新训练

## 六、回测结果有效性验证

> 应对高夏普（9.53~27.75）的过拟合怀疑，需要在独立样本外数据上交叉验证。

### 验证方法

**方法一 ⭐：时间序列交叉验证（Rolling OOS）** — 优先执行

将 OOS 期（2021-03~2024-03）分为 3 个子区间独立回测：

| 子区间 | 期望 | 若否则 |
|--------|------|--------|
| 2021-03 ~ 2022-03 | 夏普 > 2 | 夏普 < 0 → 过拟合 |
| 2022-03 ~ 2023-03 | 夏普 > 2 | 同上 |
| 2023-03 ~ 2024-03 | 夏普 > 2 | 同上 |

3 个区间均夏普 > 2 → 模型信号在不同市场有效。预计耗时 **5 分钟**。

**方法二：随机标签置换检验（Permutation Test）**

随机打乱预测值到标的的映射，运行 1000 次回测。真实夏普 > 99.9% 随机夏普 → 通过。预计 **30 分钟**。

**方法三：纯随机策略基线**

生成同数量随机入场信号，运行 1000 次模拟。真实夏普 > 95% 随机夏普 → 非运气。预计 **10 分钟**。

### 接受标准

- ✅ **通过**: 方法一全部 3 区间夏普 > 2，且方法三 p-value < 0.05
- ⚠️ **存疑**: 方法一通过但方法三不通过
- ❌ **失败**: 方法一任何区间夏普 < 0
